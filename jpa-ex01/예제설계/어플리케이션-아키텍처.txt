계층형 구조 사용
- controller, web: 웹 계층
- service: 비즈니스 로직, 트랜잭션 처리
- repository: JPA를 직접 사용하는 계층, 엔티티 매니저 사용
- domain: 엔티티가 모여 있는 계층, 모든 계층에서 사용

패키지 구조 (jpabook.jpashop)
com.minod.shop
 .domain
 .exception
 .repository
 .service
 .web

        상품 서비스 개발
    1-3. 주문 도메인 개발

        주문 서비스 개발 ㅇ
        주문 검색 기능 개발 (동적쿼리?? 날짜기준, 회원기준 검색 할 수 있어야함.)
